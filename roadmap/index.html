Product Roadmap

From proof-of-concept to production-ready Configuration-as-Code. See what's coming next for Typeflows.

Typeflows Development Approach

The timeline is driven by user feedback cycles, technical complexity, and development capacity. Each phase expands from workflows to complete GitHub repository configuration management, ensuring stability while delivering comprehensive .github folder automation.

Proof of Concept

COMPLETED

Core foundation with basic workflow generation and validation capabilities.

  • Basic DSL to model GitHub Workflows, Actions and config
  • Beta JVM DSL (Kotlin/Java)
  • Local syntax validation
  • Core action type definitions

Early Access

IN PROGRESS

Complete repository configuration management with visualisation.

  • JVM DSL with Gradle and Maven build integration
  • Complete .github/ folder and repository configuration
  • Workflow visualisation and marketplace actions
  • AI assistant configuration support
  • Package-based distribution and sharing
  • Beta TypeScript and Python support
  • Basic testing framework

Beta

Q1 2026

Testing capabilities and multi-language support.

  • Event-driven workflow testing framework
  • Stable TypeScript and Python DSLs
  • Multi-workflow coordination testing
  • Security policy validation

Version 1.0.0

Q2 2026

Production-ready with advanced security and orchestration.

  • Advanced workflow orchestration
  • All major languages supported
  • Workflow security vulnerability detection
  • Compliance and governance features

Future

2026+

Ecosystem expansion and platform integrations based on community feedback.

  • Discovery portal & community contributions
  • Additional language support (Go, Rust, C#)
  • GitLab, Bitbucket, and Azure DevOps support
