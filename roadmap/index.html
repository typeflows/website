Our Development Approach

Our timeline is driven by user feedback cycles, technical complexity, and team capacity. Each phase builds on proven foundations from the previous release, ensuring stability while delivering meaningful value to developers.

Proof of Concept

IN PROGRESS

Building and validating the core concept with initial prototypes.

  • DSL/conceptual design
  • Workflow YAML import/export
  • Early feedback collection

Early Access

Summer 2025

First usable version with core functionality for early adopters.

  • Stable JVM DSL
  • Basic CLI tooling
  • Simple workflow visualisations
  • Simple workflow testing

Beta

Q3 2025

Multi-language support and enhanced developer experience.

  • TypeScript and Python DSLs
  • Workflow orchestration testing
  • Workflow visualisations API

Version 1.0

Q4 2025

Production-ready release with enterprise features and stability focus.

  • Production stability and reliability
  • YAML workflow migration tooling
  • Pre-built GitHub Marketplace Actions libraries
  • Publish workflows/actions as libraries

Future

2026+

Long-term possibilities depending on adoption and community feedback.

  • Further language support (Rust, Go, Ruby)
  • GitLab CI/CD support
  • Cross-CICD system migrations
  • Workflow security scanning
  • AI-powered optimisation suggestions
